\doxysubsection{Detailed Description}
Render 2D/3D occupancy grids and algorithm paths from run outputs. 

Provides convenient plotting for benchmark runs. For 2D grids, writes a PNG with obstacles, start/goal markers, and optional path overlays. For 3D grids, writes an interactive Plotly HTML (isosurface + point cloud + paths), and can optionally save a Matplotlib 3D PNG snapshot when available.

\begin{DoxyNote}{Note}

\begin{DoxyItemize}
\item Plotly is preferred for 3D (opens in a browser). If Plotly is unavailable, the code attempts a Matplotlib 3D fallback when possible.
\item Obstacles are rendered as black (2D) or red (3D). Paths are colored and labeled by algorithm name.
\item The function auto-\/creates {\ttfamily results/viz/} if it does not exist.
\item Start/\+Goal are ALWAYS taken from the first algorithm\textquotesingle{}s iter file.
\item The GRID is ALWAYS taken from {\ttfamily configs/generate\+\_\+grid/\{env\+Name\}/\{env\+Name\}\+\_\+grid.\+json}, where {\ttfamily env\+Name} is read from the first algorithm\textquotesingle{}s iter file ({\ttfamily env\+Info.\+env\+Name}) and lowercased.
\end{DoxyItemize}
\end{DoxyNote}
