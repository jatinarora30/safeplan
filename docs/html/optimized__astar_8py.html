Detailed Description

Optimized A* planner for N-dimensional occupancy grids.

Implements an A* planner with additional path-quality and safety refinements on an N-D NumPy occupancy grid where value 0 denotes free space and value 1 denotes an obstacle. The planner accepts start and goal grid indices and returns a collision-free path if one exists.

Key features beyond a standard A*:

  • Turn-aware cost: adds a penalty based on local change in heading to discourage sharp turns (see turnHeuristics()).
  • Safety inflation: uses a distance transform to nudge path points away from nearby obstacles (see safeInflate()).
  • Path shortcutting: performs forward and backward line-of-sight pruning to reduce unnecessary waypoints (see directionalOptimize() and bidirectionalOptimize()).
  • Local edge collision checking: samples points along a segment and rejects it if any samples fall out of bounds or on obstacles within a safety radius (see isEdgeFree()).

Heuristic and neighborhood generation are provided by heuristics() and adjacentCoordinates(). Ensure the heuristic is compatible with the allowed moves and step costs.

Constructor Arguments
  • turnPenaltyCoefficients : float — multiplier for penalizing local turning
  • safetyDistGridRadius : float — minimum desired clearance from obstacles in grid cells
  • maxInflateIter : int — maximum iterations for safety inflation per waypoint
  • pointSamples : int — number of samples along an edge for collision checking
Inputs (to @ref plan())
  • start : tuple[int, ...] — start grid cell (for example, (row, col))
  • goal : tuple[int, ...] — goal grid cell
  • grid : numpy.ndarray (N-D), values {0=free, 1=obstacle}
Outputs
  • success : int — 1 if a path is found, else 0
  • path : list[tuple[int, ...]] — sequence of cells from start to goal (inclusive)
  • info : list[str] — diagnostics (for example, "Invalid goal")
Note
  • Coordinates are handled in grid index space. Steered or optimized points are rounded to integer indices before validation.
  • The distance transform is computed once per plan call and reused in collision checks and safety inflation.
  • The neighborhood includes all immediate offsets except the zero vector.
  • Only axis-aligned moves are expanded in the inner loop of plan() to keep turning penalties predictable; adjust as needed for your benchmark.
