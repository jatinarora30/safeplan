1 """
2 @file weighted_astar.py
3 @brief Weighted A* planner for N-dimensional occupancy grids.
4 
5 @details
6 Implements Weighted A* on an N-D NumPy occupancy grid where value 0 denotes free
7 space and value 1 denotes an obstacle. The planner accepts start and goal grid
8 indices and returns a collision-free path if one exists. Weighted A* scales the
9 heuristic by a user-specified weight to favor greedy expansions and reduce
10 runtime at the cost of potential suboptimality when the weight exceeds 1.
11 
12 This implementation expands a full 3^N - 1 neighborhood (axis-aligned and
13 diagonal moves). Axis-aligned steps use unit cost; off-axis steps use the
14 Euclidean length of the step vector. A priority queue orders expansions by
15 g(n) + w·h(n), where g(n) is the accumulated path cost and h(n) is the chosen
16 heuristic (Manhattan in index space).
17 
18 Neighborhood generation is provided by @ref adjacentCoordinates().
19 
20 @par Constructor Arguments
21 - @p weight : float — multiplier applied to the heuristic (w ≥ 0).
22  - w = 1 behaves like standard A* with this heuristic.
23  - w > 1 is greedier and usually faster but may yield a longer path.
24  - w < 1 is more conservative and may expand more nodes.
25 
26 @par Inputs (to @ref plan())
27 - @p start : tuple[int, ...] — start grid cell (for example, (row, col))
28 - @p goal : tuple[int, ...] — goal grid cell
29 - @p grid : numpy.ndarray (N-D), values {0=free, 1=obstacle}
30 
31 @par Outputs
32 - @p success : int — 1 if a path is found, else 0
33 - @p path : list[tuple[int, ...]] — sequence of cells from start to goal (inclusive)
34 - @p info : list[str] — diagnostics (for example, "Invalid goal")
35 
36 @note
37 - Coordinates are handled in grid index space.
38 - Step costs: axis steps cost 1.0; diagonal/off-axis steps use the Euclidean
39  length of the index step.
40 - The neighborhood includes all immediate offsets except the zero vector.
41 - Heuristic used here is Manhattan distance; adjust if you change the move set.
42 - With w > 1, the solution is not guaranteed to be optimal.
43
