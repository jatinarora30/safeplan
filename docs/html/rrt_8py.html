Detailed Description

Rapidly-exploring Random Tree (RRT) planner for N-dimensional occupancy grids.

Implements a basic RRT over an N-D NumPy occupancy grid where value 0 denotes free space and value 1 denotes an obstacle. The planner incrementally grows a tree from the start toward random samples (with goal bias). At each step, it adds a new node no farther than the configured step size from the nearest existing node, provided the local edge is collision-free. A path is returned once the tree connects to the goal via a collision-free edge.

Collision checking (see isEdgeFree()) samples a fixed number of points along a candidate segment, rounds them to the nearest grid indices, and rejects the edge if any sampled index is out of bounds or lies on an obstacle cell.

Constructor Arguments
  • maxIter : int — maximum number of RRT expansion iterations
  • goalSampleRate : float in [0,1] — probability of directly sampling the goal
  • stepSize : float — maximum extension per step in grid index units
  • pointSamples : int — number of samples along an edge for collision checking
Inputs (to @ref plan())
  • start : tuple[int, ...] — start grid cell (for example, (row, col))
  • goal : tuple[int, ...] — goal grid cell
  • grid : numpy.ndarray (N-D), values {0=free, 1=obstacle}
Outputs
  • success : int — 1 if a path is found, else 0
  • path : list[tuple[int, ...]] — sequence of cells from start to goal (inclusive)
  • info : list[str] — diagnostics (for example, "Invalid start", "Goal has obstacle", "Ran out of max iterations couldn't find path")
Note
  • Coordinates are treated in grid index space. Steered points are rounded to integer indices before validation and insertion.
  • The effective step during steering is clamped by the minimum of the step size and the straight-line distance between start and goal to avoid overshoot on very small problems.
  • Edge feasibility depends on pointSamples; too few samples may miss thin obstacles, while too many may slow planning.
  • Nearest neighbor is a linear scan; for large trees, consider a spatial index such as a kd-tree or ball-tree.
  • RRT is probabilistically complete but not optimal; use RRT* for asymptotic optimality (not implemented here).

@complexity Per linear in the number of current tree nodes for nearest neighbor, plus linear in the number of edge samples for collision checking. Worst case is proportional to the maximum number of iterations multiplied by the sum of those two costs.
