1 """
2 @file cbf_rrt.py
3 @brief Rapidly-exploring Random Tree with Control Barrier Functions (CBF-RRT) for N-D occupancy grids.
4 
5 @details
6 Implements a CBF-augmented Rapidly-exploring Random Tree (RRT) over an N-D NumPy
7 occupancy grid where value 0 denotes free space and value 1 denotes an obstacle.
8 The planner grows a tree from the start toward random samples (with an adjustable
9 goal bias). At each step, it proposes a new node no farther than the configured
10 step size from the nearest existing node. The new node is accepted only if the
11 local edge is collision-free and a CBF-based safety check passes. A path is
12 returned once the tree connects to the goal with a collision-free edge.
13 
14 Collision checking (see @ref isEdgeFree()) samples a fixed number of points along
15 a candidate segment, rounds them to the nearest grid indices, and rejects the
16 edge if any sampled index is out of bounds or lies on an obstacle.
17 
18 The method @ref getObstacleCentersRadii() computes approximate obstacle centers
19 and radii from connected components. These are used by a grid-based CBF safety
20 test in @ref isSafe().
21 
22 @par Constructor Arguments
23 - @p maxIter : int — maximum number of RRT expansion iterations
24 - @p goalSampleRate : float in [0,1] — probability of directly sampling the goal
25 - @p stepSize : float — maximum extension per step in grid index units
26 - @p pointSamples : int — number of samples along an edge for collision checking
27 - @p gamma1 : float — CBF parameter (first-order coefficient)
28 - @p gamma2 : float — CBF parameter (second-order coefficient)
29 
30 @par Inputs (to @ref plan())
31 - @p start : tuple[int, ...] — start grid cell (for example, (row, col))
32 - @p goal : tuple[int, ...] — goal grid cell
33 - @p grid : numpy.ndarray (N-D), values {0=free, 1=obstacle}
34 
35 @par Outputs
36 - @p success : int — 1 if a path is found, else 0
37 - @p path : list[tuple[int, ...]] — sequence of cells from start to goal (inclusive)
38 - @p info : list[str] — diagnostics (for example, "Invalid start", "Goal has obstacle",
39  "Ran out of max iterations couldn't find path")
40 
41 @note
42 - Coordinates are in grid index space. Steered points are rounded to integer
43  indices before validation and insertion.
44 - The step used during steering is limited by the configured step size and by
45  the straight-line distance between start and goal to avoid overshoot on very
46  small problems.
47 - Edge feasibility depends on @p pointSamples; too few may miss thin obstacles,
48  too many may slow planning.
49 - Nearest neighbor is implemented as a linear scan; for large trees, consider a
50  spatial index such as a kd-tree or ball-tree.
51 - RRT is probabilistically complete but not optimal. Use RRT* variants for
52  asymptotic optimality (not implemented here).
53 - The CBF safety check is an approximation derived from obstacle components.
54 
55 @complexity
56 Per iteration: linear in the number of existing tree nodes for nearest neighbor,
57 plus linear in the number of edge samples for collision checking.
58 Worst case is proportional to the maximum number of iterations multiplied by the
59 sum of those two costs.
60 
61 @see BasePlanner, AStar
62 
63 @references
64 - @cite CBF_RRT_Hsu2020 Hsu et al., "Safe Path Planning Using Control Barrier Functions,"
65  arXiv:2011.06748. https://arxiv.org/pdf/2011.06748

75  @class CBFRRT
76  @brief RRT planner with Control Barrier Function (CBF) safety checks.
77 
78  @details
79  The planner maintains a tree of collision-free grid indices. Each iteration:
80  1) Sample a random node with goal bias (see @ref getRandomNode()).
81  2) Find the nearest existing node in the tree (see @ref getNearestNode()).
82  3) Steer from the nearest node toward the random sample by at most the step size.
83  4) Accept the new node only if both the edge is collision-free (see @ref isEdgeFree())
84  and the CBF safety condition passes (see @ref isSafe()).
85  5) If the new node is close enough to the goal and the final edge is free,
86  declare success and reconstruct the path.

209  def getObstacleCentersRadii(self):
210  """
211  @brief Find centers and radii of connected obstacle components.
212  @details
213  Labels connected components of obstacle cells. For each component:
214  the center is the mean of its cell indices (centroid), and the radius
215  is the largest Euclidean distance from this centroid to any cell in
216  the component. These values provide a coarse approximation of obstacle
217  geometry for use in the CBF check.
218  @return (numpy.ndarray, numpy.ndarray)
219  An array of centers with shape (num_components, dimension) and
220  an array of radii with shape (num_components,).
221  """
222  labeled_grid, num_features = label(self.gridgrid)
223  centers, radii = [], []
224  for label_num in range(1, num_features + 1):
225  positions = np.argwhere(labeled_grid == label_num) # cells in this obstacle
226  center = positions.mean(axis=0) # centroid
227  # compute radius as farthest cell from centroid
228  if len(positions) > 0:
229  dists = np.linalg.norm(positions - center, axis=1)
230  radius = dists.max()
231  else:
232  radius = 0.0
233  centers.append(center)
234  radii.append(radius)
235 
236  return np.array(centers), np.array(radii)

252  """
253  @brief Control Barrier Function (CBF) safety check for a candidate cell and velocity.
254  @details
255  Uses the obstacle component centers and radii computed by @ref getObstacleCentersRadii().
256  For each obstacle, a barrier value is formed from the squared distance to the
257  obstacle center minus the squared radius. A change rate is computed using the
258  steer vector. Two user-supplied parameters (gamma1 and gamma2) combine these
259  quantities into a safety measure. If the measure indicates increasing safety
260  or adequate safety margin for any obstacle component, the candidate state is
261  considered safe for insertion into the tree.
262  @param cell tuple[int, ...] Candidate grid cell.
263  @param vel sequence[float] Steer vector used to reach the candidate cell.
264  @return bool True if the CBF condition passes; otherwise False.
265  @note This is an approximate, grid-based CBF that uses component centroids and radii.
266  """

271  b1Dot=self.gamma1gamma1*bDot+2*np.sum(np.square(vel))
