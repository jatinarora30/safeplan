1 """
2 @file rrt.py
3 @brief Rapidly-exploring Random Tree (RRT) planner for N-dimensional occupancy grids.
4 
5 @details
6 Implements a basic RRT over an N-D NumPy occupancy grid where value 0 denotes
7 free space and value 1 denotes an obstacle. The planner incrementally grows a
8 tree from the start toward random samples (with goal bias). At each step, it
9 adds a new node no farther than the configured step size from the nearest
10 existing node, provided the local edge is collision-free. A path is returned
11 once the tree connects to the goal via a collision-free edge.
12 
13 Collision checking (see @ref isEdgeFree()) samples a fixed number of points
14 along a candidate segment, rounds them to the nearest grid indices, and rejects
15 the edge if any sampled index is out of bounds or lies on an obstacle cell.
16 
17 @par Constructor Arguments
18 - @p maxIter : int — maximum number of RRT expansion iterations
19 - @p goalSampleRate : float in [0,1] — probability of directly sampling the goal
20 - @p stepSize : float — maximum extension per step in grid index units
21 - @p pointSamples : int — number of samples along an edge for collision checking
22 
23 @par Inputs (to @ref plan())
24 - @p start : tuple[int, ...] — start grid cell (for example, (row, col))
25 - @p goal : tuple[int, ...] — goal grid cell
26 - @p grid : numpy.ndarray (N-D), values {0=free, 1=obstacle}
27 
28 @par Outputs
29 - @p success : int — 1 if a path is found, else 0
30 - @p path : list[tuple[int, ...]] — sequence of cells from start to goal (inclusive)
31 - @p info : list[str] — diagnostics (for example, "Invalid start", "Goal has obstacle",
32  "Ran out of max iterations couldn't find path")
33 
34 @note
35 - Coordinates are treated in grid index space. Steered points are rounded to
36  integer indices before validation and insertion.
37 - The effective step during steering is clamped by the minimum of the step size
38  and the straight-line distance between start and goal to avoid overshoot on
39  very small problems.
40 - Edge feasibility depends on @p pointSamples; too few samples may miss thin
41  obstacles, while too many may slow planning.
42 - Nearest neighbor is a linear scan; for large trees, consider a spatial index
43  such as a kd-tree or ball-tree.
44 - RRT is probabilistically complete but not optimal; use RRT* for asymptotic
45  optimality (not implemented here).
46 
47 @complexity
48 Per iteration: linear in the number of current tree nodes for nearest neighbor,
49 plus linear in the number of edge samples for collision checking. Worst case is
50 proportional to the maximum number of iterations multiplied by the sum of those
51 two costs.
52

67  @brief RRT planner operating on N-D occupancy grids.
68 
69  @details
70  The planner maintains a tree of explored, collision-free grid indices. Each
71  iteration:
72  1) Sample a random node with goal bias (see @ref getRandomNode()).
73  2) Find the nearest existing node in the tree (see @ref getNearestNode()).
74  3) Steer from the nearest node toward the random sample by at most the step size.
75  4) If the edge is collision-free (see @ref isEdgeFree()), add the new node.
76  5) If the new node is close enough to the goal and the final edge is free,
77  declare success and reconstruct the path.
