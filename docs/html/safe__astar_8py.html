Detailed Description

Clearance-aware A* on N-D occupancy grids using EDT-based safety costs.

This module implements an A* planner on an N-D occupancy grid (0 = free, 1 = obstacle) that explicitly trades off path length and safety. Instead of inflating obstacles, it computes a Euclidean Distance Transform (EDT) over free space and adds a soft safety penalty to each edge cost:

g(u→v) = step_cost(u, v) + safetyFactor / (epsilon + 0.5*(EDT[u] + EDT[v]))

This penalizes moves that pass close to obstacles while keeping the full free space available (no hard removal of narrow passages). It differs from “inflated A*” where near-obstacle cells are marked occupied.

Conventions
  • Grid values: 0 = free, 1 = obstacle.
  • Coordinates: integer grid indices in N dimensions.
  • Neighborhood: all 3^N - 1 offsets (axis-aligned + diagonals).
  • Step costs: axis moves cost 1.0; diagonal/off-axis cost L2 magnitude.
  • Heuristic: Manhattan (L1) by default. If you allow diagonal moves and want strict admissibility, prefer Chebyshev (L∞) or Euclidean (L2 with consistent step costs).
Inputs (to plan())
  • start : tuple[int, ...] — start grid cell (e.g., (row, col[, ...]))
  • goal : tuple[int, ...] — goal grid cell
  • grid : numpy.ndarray (N-D), values {0=free, 1=obstacle}
Constructor Args
  • safetyFactor : float — weight on the EDT-based safety penalty (larger → more clearance-seeking)
  • epsilon : float — small positive constant to avoid division by zero in the safety term
Outputs (from plan())
  • success : int — 1 if a path is found, else 0
  • path : list[tuple[int, ...]] — sequence of cells from start→goal (inclusive) or empty
  • info : list[str] — diagnostics (e.g., "Invalid goal", "Failed to reconstruct path")
Note
  • This “soft-constraint” approach keeps narrow passages feasible but discourages them unless necessary. Tune safetyFactor and epsilon to balance optimality vs. clearance.
  • With diagonal moves, Manhattan (L1) is not strictly admissible; use L∞ or L2 with consistent step costs if optimality guarantees are required.
