Detailed Description

Aggregate, compare, and visualize benchmark statistics from JSON outputs.

Loads a run configuration (algorithms, evaluations, and run metadata), scans per-algorithm JSON outputs produced under a run directory, computes averaged metrics, and optionally renders a table figure. Results are saved as a JSON summary (and PNG if requested).

Directory conventions:

  • Outputs are expected under outputs/<runDetails>/<algo>/*.json.
  • Aggregated stats are written to results/<runDetails>Stats.json.
  • Optional figure is saved to results/<runDetails>.png.

Evaluation handling:

  • The code builds a consistent evaluation key list (self.evals) from the config file and averages each metric across all JSON files found for a given algorithm.
  • Keys matching a generic “success rate” pattern (e.g., SuccessRate, success_rate, success) are normalized by the global maximum iteration count across algorithms; all other metrics are averaged over the number of files found for that algorithm.
Expected Config Schema (JSON at runConfigPath)
  • algoDetails : list of objects with at least a name field for each algorithm.
  • evalDetails : list of objects with at least a name field for each metric.
  • runDetails : string run identifier (used as subdirectory under outputs/).
Outputs
  • Printed pandas DataFrame of averaged metrics (row = algorithm, col = metric).
  • JSON summary written to results/ <runDetails>Stats.json.
  • Optional PNG table at results/ <runDetails>.png when saveStatsImage is True.
Note
  • Missing or malformed result files are skipped with a console message.
  • If an algorithm has no result files, its metrics are set to NaN in the output.
  • The module creates a results/ directory if it does not already exist.

pandas.DataFrame, matplotlib, pandas.plotting.table
