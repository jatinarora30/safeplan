1 """
2 @file optimized_astar.py
3 @brief Optimized A* planner for N-dimensional occupancy grids.
4 
5 @details
6 Implements an A* planner with additional path-quality and safety refinements on
7 an N-D NumPy occupancy grid where value 0 denotes free space and value 1 denotes
8 an obstacle. The planner accepts start and goal grid indices and returns a
9 collision-free path if one exists.
10 
11 Key features beyond a standard A*:
12 - Turn-aware cost: adds a penalty based on local change in heading to discourage
13  sharp turns (see @ref turnHeuristics()).
14 - Safety inflation: uses a distance transform to nudge path points away from
15  nearby obstacles (see @ref safeInflate()).
16 - Path shortcutting: performs forward and backward line-of-sight pruning to
17  reduce unnecessary waypoints (see @ref directionalOptimize() and
18  @ref bidirectionalOptimize()).
19 - Local edge collision checking: samples points along a segment and rejects it if
20  any samples fall out of bounds or on obstacles within a safety radius
21  (see @ref isEdgeFree()).
22 
23 Heuristic and neighborhood generation are provided by @ref heuristics() and
24 @ref adjacentCoordinates(). Ensure the heuristic is compatible with the allowed
25 moves and step costs.
26 
27 @par Constructor Arguments
28 - @p turnPenaltyCoefficients : float — multiplier for penalizing local turning
29 - @p safetyDistGridRadius : float — minimum desired clearance from obstacles in grid cells
30 - @p maxInflateIter : int — maximum iterations for safety inflation per waypoint
31 - @p pointSamples : int — number of samples along an edge for collision checking
32 
33 @par Inputs (to @ref plan())
34 - @p start : tuple[int, ...] — start grid cell (for example, (row, col))
35 - @p goal : tuple[int, ...] — goal grid cell
36 - @p grid : numpy.ndarray (N-D), values {0=free, 1=obstacle}
37 
38 @par Outputs
39 - @p success : int — 1 if a path is found, else 0
40 - @p path : list[tuple[int, ...]] — sequence of cells from start to goal (inclusive)
41 - @p info : list[str] — diagnostics (for example, "Invalid goal")
42 
43 @note
44 - Coordinates are handled in grid index space. Steered or optimized points are
45  rounded to integer indices before validation.
46 - The distance transform is computed once per plan call and reused in collision
47  checks and safety inflation.
48 - The neighborhood includes all immediate offsets except the zero vector.
49 - Only axis-aligned moves are expanded in the inner loop of @ref plan() to keep
50  turning penalties predictable; adjust as needed for your benchmark.
51 
52 @see BasePlanner
