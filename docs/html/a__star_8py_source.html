2 @file a_star.py
3 @brief N-dimensional A* (A-star) planner over NumPy occupancy grids.
4 
5 @details
6 This module implements A* search on an N-D occupancy grid where `0` denotes free
7 space and `1` denotes an obstacle. The planner accepts start and goal grid indices
8 (tuples of ints) and returns a collision-free path if one exists.
9 
10 - Heuristic: By default, this implementation uses the **Manhattan (L1)** distance.
11 - Neighborhood: All 3^N - 1 offsets are considered (axis-aligned and diagonal moves).
12 - Costs: Axis-aligned steps cost 1.0; diagonal/off-axis steps use the L2 magnitude
13  of the step vector.
14 
15 @attention Heuristic admissibility depends on the move set. If diagonal moves are
16 allowed (as in this implementation), Manhattan distance is **not** strictly
17 admissible unless you adjust step costs accordingly. Prefer **Chebyshev** (L∞) or
18 Euclidean (L2 with consistent step costs) for diagonal-enabled neighborhoods.
19 
20 @par Inputs
21 - @p start : tuple[int, ...] — start grid cell (e.g., (row, col, ...))
22 - @p goal : tuple[int, ...] — goal grid cell
23 - @p grid : numpy.ndarray (N-D), values {0=free, 1=obstacle}
24 
25 @par Outputs
26 - @p success : int — 1 if a path is found, else 0
27 - @p path : list[tuple[int, ...]] — cells from start→goal (inclusive) or empty if none
28 - @p info : list[str] — diagnostics (e.g., "Invalid goal")
29 
30 @note Ensure the heuristic matches the move set:
31  - 4-neighbors (axis-only) → Manhattan (L1)
32  - 8-neighbors / diagonal moves → Chebyshev (L∞) or Euclidean (L2 with proper step costs)
33 
34 @see BasePlanner
