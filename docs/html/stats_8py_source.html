2 @file stats.py
3 @brief Aggregate, compare, and visualize benchmark statistics from JSON outputs.
4 
5 @details
6 Loads a run configuration (algorithms, evaluations, and run metadata), scans
7 per-algorithm JSON outputs produced under a run directory, computes averaged
8 metrics, and optionally renders a table figure. Results are saved as a JSON
9 summary (and PNG if requested).
10 
11 Directory conventions:
12 - Outputs are expected under `outputs/<runDetails>/<algo>/*.json`.
13 - Aggregated stats are written to `results/<runDetails>Stats.json`.
14 - Optional figure is saved to `results/<runDetails>.png`.
15 
16 Evaluation handling:
17 - The code builds a consistent evaluation key list (`self.evals`) from the
18  config file and averages each metric across all JSON files found for a given
19  algorithm.
20 - Keys matching a generic “success rate” pattern (e.g., `SuccessRate`,
21  `success_rate`, `success`) are normalized by the **global** maximum iteration
22  count across algorithms; all other metrics are averaged over the number of
23  files found for that algorithm.
24 
25 @par Expected Config Schema (JSON at @p runConfigPath)
26 - @c algoDetails : list of objects with at least a @c name field for each algorithm.
27 - @c evalDetails : list of objects with at least a @c name field for each metric.
28 - @c runDetails : string run identifier (used as subdirectory under @c outputs/).
29 
30 @par Outputs
31 - Printed pandas DataFrame of averaged metrics (row = algorithm, col = metric).
32 - JSON summary written to @c results/<runDetails>Stats.json.
33 - Optional PNG table at @c results/<runDetails>.png when @p saveStatsImage is True.
34 
35 @note
36 - Missing or malformed result files are skipped with a console message.
37 - If an algorithm has no result files, its metrics are set to NaN in the output.
38 - The module creates a @c results/ directory if it does not already exist.
39 
40 @see pandas.DataFrame, matplotlib, pandas.plotting.table
41 """

159  def compute(self):
160  """
161  @brief Aggregate metrics per algorithm and write summary artifacts.
162 
163  @details
164  For each algorithm directory under the run path, all JSON result files are
165  loaded and their @c evaluation blocks are accumulated.
166 
167  - For keys deemed “success rate” (case/format variations accepted),
168  we treat each iteration as a 0/1 success indicator, pad missing
169  iterations with zeros up to @c self.max_iterations, and compute:
170  mean = total_successes / max_iterations
171  std = std dev over max_iterations 0/1 samples.
172  - For all other metrics, mean and std are computed over the per-file
173  values available for that algorithm.
174 
175  Two DataFrames are printed: means and standard deviations.
176  A JSON summary is written to results/<runDetails>Stats.json with
177  structure:
178  { algo: { metric: { "mean": ..., "std": ... }, ... }, ... }
179 
180  If @c saveStatsImage is True, a PNG table of "mean±std" is also saved.
181  """
