Detailed Description

Weighted A* planner for N-dimensional occupancy grids.

Implements Weighted A* on an N-D NumPy occupancy grid where value 0 denotes free space and value 1 denotes an obstacle. The planner accepts start and goal grid indices and returns a collision-free path if one exists. Weighted A* scales the heuristic by a user-specified weight to favor greedy expansions and reduce runtime at the cost of potential suboptimality when the weight exceeds 1.

This implementation expands a full 3^N - 1 neighborhood (axis-aligned and diagonal moves). Axis-aligned steps use unit cost; off-axis steps use the Euclidean length of the step vector. A priority queue orders expansions by g(n) + w·h(n), where g(n) is the accumulated path cost and h(n) is the chosen heuristic (Manhattan in index space).

Neighborhood generation is provided by adjacentCoordinates().

Constructor Arguments
  • weight : float — multiplier applied to the heuristic (w ≥ 0).
    • w = 1 behaves like standard A* with this heuristic.
    • w > 1 is greedier and usually faster but may yield a longer path.
    • w < 1 is more conservative and may expand more nodes.
Inputs (to @ref plan())
  • start : tuple[int, ...] — start grid cell (for example, (row, col))
  • goal : tuple[int, ...] — goal grid cell
  • grid : numpy.ndarray (N-D), values {0=free, 1=obstacle}
Outputs
  • success : int — 1 if a path is found, else 0
  • path : list[tuple[int, ...]] — sequence of cells from start to goal (inclusive)
  • info : list[str] — diagnostics (for example, "Invalid goal")
Note
  • Coordinates are handled in grid index space.
  • Step costs: axis steps cost 1.0; diagonal/off-axis steps use the Euclidean length of the index step.
  • The neighborhood includes all immediate offsets except the zero vector.
  • Heuristic used here is Manhattan distance; adjust if you change the move set.
  • With w > 1, the solution is not guaranteed to be optimal.
